My perspective on using AI in programming has changed significantly throughout taking this course. Before this course, I would often paste an entire problem into an AI tool and let it generate the solution for me, which meant I was not truly learning or understanding the underlying concepts. This course taught me how to use AI more effectively and intentionally, as I would break problems down into smaller parts and I would ask AI to explain concepts instead of producing the code for it. Doing this made a huge difference in my understanding, as I was able to walk through implementations on my own and actually understand the code I was writing. 

Going forward, I plan to continue using AI as a learning aid instead of a shortcut. Instead of relying on it to write code for me, I will use it to clarify concepts I am struggling with, help me understand error messages when debugging, and verify my reasoning as I work through a problem step by step. Not only does this approach make me a better programmer, it also ensures that I am building skills I can apply independently with AI being used as a guide instead of having it do all the work.
